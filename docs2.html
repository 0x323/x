1. Overview

CheckInPoint adalah sistem token gamifikasi di Sepolia Base. Pengguna mendapatkan P voluntarily melalui check-in harian, referral, dan taruhan.

  • Check-In Harian: Dapatkan 1 Poin per hari.
  • Program Referral: Dapatkan 5 Poin per taruhan yang dibuat oleh referee.
  • Taruhan: Pasang taruhan, menangkan hadiah berdasarkan kecocokan prefiks angka.

Token Point memiliki 18 desimal. Transfer dinonaktifkan untuk mengontrol distribusi.

4. Check-In System

Pengguna dapat check-in sekali per hari UTC untuk mendapatkan 1 Poin. Sistem mencegah check-in ganda dalam hari yang sama.

Fungsi Utama

Fungsi checkIn memeriksa apakah pengguna sudah check-in hari ini, mencetak 1 Poin jika belum, memperbarui hari check-in terakhir, dan mencatat event.

function checkIn() public {
    uint256 currentDayUTC = block.timestamp / 1 days;
    require(_lastCheckIn[msg.sender] < currentDayUTC, "You have already checked in today");
    _mint(msg.sender, POINTS_PER_CHECKIN);
    _lastCheckIn[msg.sender] = currentDayUTC;
    checkInHistory[msg.sender].push(CheckInLog({
        user: msg.sender,
        dayUTC: currentDayUTC,
        timestamp: block.timestamp,
        reward: POINTS_PER_CHECKIN
    }));
    emit CheckIn(msg.sender, POINTS_PER_CHECKIN, block.timestamp);
    emit CheckInLogCreated(msg.sender, currentDayUTC, block.timestamp, POINTS_PER_CHECKIN);
}
  • Logika: Memverifikasi batas check-in harian, mencetak 1 Poin, dan mencatat event.
  • Event: CheckIn, CheckInLogCreated
  • Batasan: Satu check-in per hari UTC.

Contoh

Kode JavaScript ini menggunakan web3.js untuk memanggil fungsi checkIn, memungkinkan pengguna check-in dan mendapatkan 1 Poin.

const contract = new web3.eth.Contract(abi, contractAddress);
await contract.methods.checkIn().send({ from: userAddress });

Fungsi Tampilan

  • getCheckInLog(address user, uint256 index): Mengambil log check-in tertentu.
  • getCheckInCount(address user): Mengembalikan jumlah total check-in.

5. Referral System

Pengguna dapat mengatur ID referral unik dan mendapatkan 5 Poin ketika referee mereka memasang taruhan.

Fungsi Utama

setReferralId menetapkan ID referral unik untuk pengguna, dan setReferralById menghubungkan pengguna ke referrer, memastikan keunikan dan mencegah self-referral.

function setReferralId(string memory referralId) public {
    require(bytes(referralIdOf[msg.sender]).length == 0, "You have already set a referral ID");
    require(userByReferralId[referralId] == address(0), "Referral ID already taken");
    referralIdOf[msg.sender] = referralId;
    userByReferralId[referralId] = msg.sender;
    emit ReferralIdSet(msg.sender, referralId);
}
function setReferralById(string memory referralId) public {
    require(referredBy[msg.sender] == address(0), "You have already been referred");
    address referrer = userByReferralId[referralId];
    require(referrer != address(0), "Referral ID not valid");
    require(referrer != msg.sender, "You cannot refer yourself");
    referredBy[msg.sender] = referrer;
    referralInfo[msg.sender].referrerId = referralId;
    referees[referrer].push(msg.sender);
    emit Referred(msg.sender, referrer, referralId);
}
  • Batasan:
    • ID referral harus unik.
    • Tidak boleh mereferensikan diri sendiri.
    • ID referral hanya bisa diatur sekali.
  • Event: ReferralIdSet, Referred, ReferralReward

Contoh

Kode ini mengatur ID referral untuk pengguna dan menghubungkan pengguna lain sebagai referee menggunakan web3.js.

const contract = new web3.eth.Contract(abi, contractAddress);
await contract.methods.setReferralId("myid123").send({ from: userAddress });
await contract.methods.setReferralById("myid123").send({ from: refereeAddress });

Fungsi Tampilan

  • getRefereeCount(address referrer): Mengembalikan jumlah referee.
  • getReferees(address referrer): Mengembalikan daftar alamat referee.

6. Betting System

Pengguna dapat memasang taruhan sekali per hari dengan nomor (1-20 karakter), pengali (1-10), dan jumlah pembayaran. Setiap taruhan berbiaya 1 Poin per instance, dengan hadiah berdasarkan jumlah pembayaran dan bonus untuk nomor yang dimulai dengan "00" atau "01".

Fungsi Utama

Fungsi placeBet memvalidasi input, mengurangi biaya taruhan, mencetak hadiah, mencatat taruhan, dan memberikan hadiah referral jika berlaku.

function placeBet(string memory _number, uint256 _times, uint256 _payoutAmount) public { 
    require(bytes(_number).length >= 1 && bytes(_number).length <= 20, "Invalid number input");
    require(_times > 0 && _times <= 10, "Bet times must be between 1 and 10");
    require(_payoutAmount > 0, "Invalid payout amount");
    string memory frontDigits = substring(_number, 0, 2);
    if (compareStrings(frontDigits, "00") || compareStrings(frontDigits, "01")) {
        _payoutAmount += 10 * (10 ** 18);
    }
    uint256 currentDayUTC = block.timestamp / 1 days;
    require(_lastBetDayUTC[msg.sender] < currentDayUTC, "You can only place bet once per day");
    uint256 totalCost = BET_COST * _times;
    uint256 totalReward = _payoutAmount * (10 ** 18);
    require(_balances[msg.sender] >= totalCost, "Insufficient token to place bet");
    _balances[msg.sender] -= totalCost;
    emit Transfer(msg.sender, address(0), totalCost);
    _mint(msg.sender, totalReward);
    _lastBetDayUTC[msg.sender] = currentDayUTC;
    Bet memory newBet = Bet({
        user: msg.sender,
        number: _number,
        cost: totalCost,
        reward: totalReward,
        timestamp: block.timestamp
    });
    betHistory.push(newBet);
    userBets[msg.sender].push(newBet);
    emit PlaceBet(msg.sender, _number, totalCost, totalReward, block.timestamp);
    address referrer = referredBy[msg.sender];
    if (referrer != address(0)) {
        _mint(referrer, REFERRAL_REWARD);
        referralRewards[referrer] += REFERRAL_REWARD;
        emit ReferralReward(referrer, msg.sender, REFERRAL_REWARD, block.timestamp);
    }
}
  • Parameter:
    • _number: String untuk taruhan (1-20 karakter).
    • _times: Pengali taruhan (1-10).
    • _payoutAmount: Hadiah dasar, meningkat 10 Poin untuk prefiks "00" atau "01".
  • Batasan: iclude
    • Satu taruhan per hari UTC.
    • Saldo cukup untuk biaya total.
    • Input nomor dan pengali valid.
  • Event: PlaceBet, Transfer, ReferralReward (jika berlaku)

Contoh

Kode ini memasang taruhan dengan nomor "01234", pengali 2, dan pembayaran 3 Poin menggunakan web3.js.

const contract = new web3.eth.Contract(abi, contractAddress);
await contract.methods.placeBet("01234", 2, 3).send({ from: userAddress });

Fungsi Tampilan

  • getUserBetCount(address user): Mengembalikan jumlah taruhan yang dipasang pengguna.
  • getUserBet(address user, uint256 index): Mengambil taruhan tertentu untuk pengguna.
  • getBetHistoryLength(): Mengembalikan jumlah total taruhan dalam riwayat global.
  • getBet(uint256 index): Mengambil taruhan tertentu dari riwayat global.

8. API Methods

Kontrak menyediakan fungsi berikut untuk interaksi:

Fungsi Token Inti

  • totalSupply() returns (uint256): Mengembalikan total pasokan token.
  • balanceOf(address account) returns (uint256): Mengembalikan saldo token akun.
  • transfer(address, uint256): Dinonaktifkan untuk mencegah transfer langsung.

Fungsi Check-In

  • checkIn(): Melakukan check-in harian.
  • getCheckInLog(address user, uint256 index) returns (CheckInLog): Mengambil log check-in.
  • getCheckInCount(address user) returns (uint256): Mengembalikan jumlah check-in.

Fungsi Referral

  • setReferralId(string referralId): Mengatur ID referral pengguna.
  • setReferralById(string referralId): Menghubungkan pengguna ke referrer.
  • getRefereeCount(address referrer) returns (uint256): Mengembalikan jumlah referee.
  • getReferees(address referrer) returns (address[]): Mengembalikan daftar referee.

Fungsi Taruhan

  • placeBet(string _number, uint256 _times, uint256 _payoutAmount): Memasang taruhan.
  • getUserBetCount(address user) returns (uint256): Mengembalikan jumlah taruhan.
  • getUserBet(address user, uint256 index) returns (Bet): Mengambil taruhan tertentu.
  • getBetHistoryLength() returns (uint256): Mengembalikan jumlah total taruhan.
  • getBet(uint256 index) returns (Bet): Mengambil taruhan dari riwayat global.

Fungsi Utilitas

  • compareStrings(string a, string b) returns (bool): Membandingkan dua string untuk kesetaraan.
  • substring(string str, uint startIndex, uint endIndex) returns (string): Mengekstrak substring.

9. Error Handling

Kontrak mencakup pemeriksaan error berikut untuk menangani aksi tidak valid:

  • "You have already checked in today": Mencegah check-in ganda dalam hari UTC yang sama.
  • "You have already set a referral ID": Memastikan ID referral hanya diatur sekali.
  • "Referral ID already taken": Memastikan keunikan ID referral.
  • "You have already been referred": Mencegah perubahan referrer.
  • "Referral ID not valid": Memastikan ID referral ada.
  • "You cannot refer yourself": Mencegah self-referral.
  • "Invalid number input": Memastikan nomor taruhan 1-20 karakter.
  • "Bet times must be between 1 and 10": Memvalidasi pengali taruhan.
  • "Invalid payout amount": Memastikan jumlah pembayaran positif.
  • "You can only place bet once per day": Membatasi taruhan satu kali per hari.
  • "Insufficient token to place bet": Memeriksa saldo cukup.
  • "Index out of bounds": Mencegah akses indeks array tidak valid.
  • "Cannot mint to zero address": Mencegah minting ke alamat nol.
  • "Transfers are not allowed": Memblokir transfer token langsung.

const Web3 = require('web3');
const web3 = new Web3('https://sepolia.base.org');
const contractAddress = 'YOUR_CONTRACT_ADDRESS';
const abi = [/* ABI JSON */];

// Check balance
const balance = await contract.methods.balanceOf(userAddress).call();
console.log(`Balance: ${web3.utils.fromWei(balance, 'ether')} Points`);
