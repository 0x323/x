9. Error Handling


The contract includes the following error checks to handle invalid actions:


  • "You have already checked in today":
    Prevents multiple check-ins in the same UTC day.

  • "You have already set a referral ID":
    Ensures referral IDs are set only once.

  • "Referral ID already taken":
    Enforces referral ID uniqueness.

  • "You have already been referred":
    Prevents changing referrers.

  • "Referral ID not valid":
    Ensures the referral ID exists.

  • "You cannot refer yourself":
    Prevents self-referrals.

  • "Invalid number input":
    Ensures bet number is 00-9999 characters.

  • "Bet times must be between 1 / 5":
    Validates bet multiplier.

  • "Invalid payout amount":
    Ensures payout amount is positive.

  • "You can only place bet once per day":
    Limits bets to one per day.

  • "Insufficient token to place bet":
    Checks for sufficient balance.

  • "Index out of bounds":
    Prevents accessing invalid array indices.

  • "Cannot mint to zero address":
    Prevents minting to the zero address.

  • "Transfers are not allowed":
    Blocks direct token transfers.
