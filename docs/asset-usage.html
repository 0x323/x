5. Asset Usage

5.1 Permitted Usage

  1. Brand and Logo:
    • Used by the Asset Owner or authorized licensees.
    • Examples: Logo on dApps, “Lotto Chain” in whitepapers, branding in NFTs.
    • Must include ™ (unregistered) or ® (registered) symbols.
  2. Source Code:
    • Restricted to the Asset Owner or explicit licensees.
    • Limited to platform operations or licensed projects.
    • Examples: Official smart contract deployment, dApp integration by partners.
  3. Non-commercial community use (e.g., tutorials) is allowed with attribution: “© Lotto Chain, all rights reserved.”

5.2 Asset Licensing

  1. License Types:
    • Brand and Logo: For branding (e.g., partner dApps).
    • Source Code: Limited licenses for code use or modification (e.g., forks for licensed projects).
  2. License Mechanisms:
    • License Smart Contract: Automates rights, duration, and royalties.
      Example smart contract function for granting a license.
      function grantLicense(address user, uint256 duration, bytes32 assetType) public onlyDAO {
    licenses[user] = License(user, assetType, block.timestamp, duration);
    emit LicenseGranted(user, assetType, duration);
}

    • Written Agreement: For strategic partners, digitally signed and hashed on the blockchain.
  3. License Terms:
    • Scope: Virtual regions, service types (e.g., lotteries, DeFi).
    • Duration: E.g., 1 year, renewable via DAO.
    • Royalties: 5-10% of revenue, paid in Lotto Chain tokens or stablecoins.
    • Prohibitions: Modifying logo/code without approval, use for illegal purposes.
  4. License Revocation: Automatic via smart contract for:
    • Failure to pay royalties.
    • Violation of guidelines (e.g., altering logo colors).
    • Harm to platform reputation.

5.3 Usage Guidelines

  1. Brand and Logo:
    • Colors: blue #007bff, blue #00b7ff.
    • Resolution: 300 dpi (digital), 1200 dpi (print).
    • Prohibited: Altering design, adding elements, use in fraudulent contexts.
  2. Source Code:
    • Must follow official documentation (e.g., smart contract ABI).
    • Modifications only for licensees, with DAO approval.
    • Attribution required: “Source code © Lotto Chain”.
  3. Full guidelines are accessible via IPFS or the official Lotto Chain website.

5.4 Non-Commercial Use

  1. Community may use brand/logo for education (e.g., blogs, videos) with attribution.
  2. Source code is not permitted for non-commercial use unless open-sourced under a specific license (e.g., MIT, approved by DAO).
  3. Harmful non-commercial use (e.g., FUD, phishing) is considered a violation.
